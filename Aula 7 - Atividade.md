# ETAPA DE AÇÃO

## Tech Rescue Challenge

Vocês foram contratados por uma empresa de tecnologia para ajudar diferentes equipes que estão enfrentando problemas em seus sistemas.

Agora vocês atuarão como uma equipe profissional de desenvolvimento de software.

Objetivos da atividade:

* analisar problemas técnicos;
* discutir soluções em equipe;
* utilizar modal verbs em inglês;
* apresentar recomendações técnicas;
* praticar comunicação profissional.

---

# Organização da Atividade

* Formem grupos de 2 integrantes;
* Cada grupo receberá um problema;
* Leiam atentamente o cenário;
* Discutam possíveis soluções;
* Produzam recomendações em inglês;
* Preparem uma breve apresentação oral.

Durante a atividade, utilizem:

* SHOULD
* MUST
* COULD
* CAN
* HAVE TO

Lembrem-se:

* os modal verbs sempre vêm antes do verbo principal;
* o verbo principal permanece na forma base.

Exemplo:

* The system should validate passwords.
* Developers must update the documentation.

# Problema 1 — Crise de Segurança

A empresa TechSafe desenvolveu uma plataforma online de gerenciamento financeiro utilizada por pequenas empresas para controlar vendas, clientes e pagamentos.

Nos últimos meses, diversos problemas de segurança começaram a acontecer:

* contas de usuários foram invadidas;
* clientes relataram acesso não autorizado aos seus dados;
* funcionários perceberam que vários usuários utilizam senhas muito simples;
* não existe confirmação de e-mail no cadastro;
* diferentes pessoas estão compartilhando a mesma conta de acesso;
* dados importantes estão armazenados sem proteção adequada.

Além disso, a empresa recebeu um alerta informando que o sistema pode sofrer ataques caso as medidas de segurança não sejam melhoradas rapidamente.

A equipe de vocês foi contratada para analisar os problemas e propor soluções de segurança para evitar futuros ataques e proteger os dados dos usuários.

# Problema 2 — Desastre no Banco de Dados

A empresa FastMarket possui um sistema de vendas online utilizado diariamente por centenas de clientes.

Recentemente, o sistema começou a apresentar sérios problemas de desempenho:

* as páginas demoram muito para carregar;
* o sistema trava em horários de maior acesso;
* algumas informações aparecem duplicadas;
* pedidos de clientes desaparecem ocasionalmente;
* o banco de dados ocupa muito mais espaço do que deveria;
* não existem backups atualizados.

A equipe técnica identificou que várias consultas SQL foram construídas sem otimização e que a estrutura do banco de dados está desorganizada.

A empresa precisa resolver esses problemas urgentemente para evitar perda de clientes e prejuízos financeiros.

Sua equipe deverá analisar a situação e propor melhorias para tornar o sistema mais rápido, seguro e organizado.

# Problema 3 — Interface Ruim

A empresa EasyTravel criou uma plataforma para compra de passagens e reservas de hotéis.

Apesar do sistema funcionar tecnicamente, os usuários estão reclamando bastante da experiência de uso:

* os botões possuem nomes confusos;
* os menus são difíceis de encontrar;
* existem informações demais na tela;
* o design muda entre páginas diferentes;
* os usuários não conseguem finalizar compras facilmente;
* muitas pessoas abandonam o sistema antes de concluir uma reserva.

A empresa percebeu que o problema não está na programação do sistema, mas sim na experiência do usuário e na organização da interface.

Sua equipe foi chamada para analisar os problemas de usabilidade e sugerir melhorias que tornem o sistema mais simples, intuitivo e agradável para os clientes.

# Problema 4 — Falha de Documentação

A empresa DevSolutions desenvolveu uma API utilizada por diferentes equipes de desenvolvimento.

Porém, a documentação técnica está extremamente incompleta:

* não existe guia de instalação;
* várias rotas da API não possuem explicação;
* faltam exemplos de requisições e respostas;
* o código não possui comentários;
* novos desenvolvedores têm dificuldade para entender o sistema;
* cada equipe documenta de uma forma diferente.

Como consequência, os desenvolvedores estão demorando muito para implementar novas funcionalidades e corrigir problemas.

A empresa precisa organizar toda a documentação técnica para melhorar a comunicação entre as equipes e facilitar o desenvolvimento do sistema.

Sua equipe deverá analisar a situação e propor soluções para melhorar a qualidade da documentação e da comunicação técnica.
