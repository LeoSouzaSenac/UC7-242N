# Tema: Modal Verbs

---


# Por que os Modal Verbs são importantes na Tecnologia?

No desenvolvimento de software, os profissionais constantemente precisam:

* sugerir melhorias;
* explicar problemas técnicos;
* definir regras e requisitos;
* comunicar-se com equipes;
* orientar usuários;
* documentar procedimentos;
* discutir soluções.

Os modal verbs são essenciais para esse tipo de comunicação.

Exemplos em contextos profissionais reais:

* The system must validate user data.
* Developers should document the API.
* We could improve application performance.
* Users cannot access admin routes.
* The team has to deploy the update today.

# O que são Modal Verbs?

Modal verbs são verbos auxiliares que modificam o significado do verbo principal.

Eles expressam:

* possibilidade;
* obrigação;
* recomendação;
* habilidade;
* permissão;
* necessidade.

Importante:

* modal verbs não mudam conforme o sujeito;
* o verbo principal permanece na forma base.

Correto:

* The team should update the server.

Incorreto:

* The team should updates the server.

---

# Principais Modal Verbs para Contextos de Tecnologia

| Modal          | Função                      | Exemplo                               |
| -------------- | --------------------------- | ------------------------------------- |
| can            | habilidade / possibilidade  | We can deploy the system today.       |
| could          | sugestão / possibilidade    | We could redesign the interface.      |
| should         | recomendação                | Users should create strong passwords. |
| must           | obrigação / regra           | Developers must test the application. |
| have to        | necessidade                 | The team has to fix the bug.          |
| cannot / can't | proibição / impossibilidade | Users can't access this page.         |

---

# Estrutura das Frases

## Forma afirmativa

Sujeito + Modal Verb + Verbo Principal

Exemplos:

* Developers should document the code.
* The system must validate input data.
* We can improve the interface.

---

## Forma negativa

Sujeito + Modal Verb + not + Verbo Principal

Exemplos:

* Users must not share passwords.
* The team should not ignore errors.
* Developers cannot deploy unstable versions.

Formas reduzidas:

* shouldn't
* can't
* mustn't

---

## Perguntas

Modal Verb + Sujeito + Verbo Principal

Exemplos:

* Can we publish the application?
* Should we change the database?
* Must users authenticate first?

# Exemplos em Contextos Profissionais

## Exemplo 1 - Reunião Técnica

Project Manager:

* The application is slow.

Developer:

* We should optimize the database queries.

Designer:

* We could simplify the interface.

Security Analyst:

* Users must create stronger passwords.

---

## Exemplo 2 - Code Review

* The function should have better names.
* Developers must document the methods.
* We could reduce duplicated code.
* This API must validate requests.

---

## Exemplo 3 - Suporte ao Usuário

* Users can't reset passwords.
* The server must be restarted.
* We should analyze the logs.
* The team has to solve this issue quickly.
