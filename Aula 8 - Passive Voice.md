# Aula 8 - Passive Voice

---

# Por que é importante?

No dia a dia de desenvolvimento, ninguém escreve documentação, changelog (é o registro das mudanças feitas em um projeto) ou bug report dizendo "eu fiz isso". O foco é sempre **o que mudou no sistema**, não quem fez.

Exemplos reais que vocês vão encontrar em qualquer empresa:

- The login page was redesigned.
- User data is encrypted automatically.
- The API was updated to version 2.0.
- Passwords are stored securely.
- This issue has been fixed.

Isso é Passive Voice (voz passiva). Hoje vamos aprender a usar.

---

# Revisão Rápida

Vocês já viram a frase ativa com Modal Verbs:

- The system must validate user data. (ativa - quem age aparece: "the system")

Hoje vamos aprender a versão passiva da mesma ideia:

- User data must be validated. (passiva - o foco é o dado, não quem valida)

---

# O que é Passive Voice?

Na voz **ativa**, o sujeito faz a ação:

> The developer fixed the bug.

Na voz **passiva**, o foco vai para quem ou o que recebe a ação. Quem fez a ação pode até desaparecer da frase:

> The bug was fixed.

Estrutura:

**Sujeito + TO BE + Verbo no Participle (V3)**

---

# Active vs Passive

| Ativa | Passiva |
|---|---|
| The team deployed the app. | The app was deployed. |
| Developers write clean code. | Clean code is written by developers. |
| The company will launch the feature. | The feature will be launched. |
| The system must validate the input. | The input must be validated. |

Note: o objeto da frase ativa ("the bug", "the app") se torna o sujeito da frase passiva.

---

# Passive Voice nos Diferentes Tempos

## Present Simple Passive

am / is / are + Participle

- The data **is encrypted**.
- Passwords **are stored** in the database.

## Past Simple Passive

was / were + Participle

- The bug **was fixed** yesterday.
- The files **were deleted** by mistake.

## Future Passive

will be + Participle

- The app **will be updated** next week.
- A new feature **will be released** soon.

## Modal + Passive

Modal + be + Participle

- The code **must be tested** before deploy.
- This bug **should be fixed** today.
- The server **can be restarted** remotely.
- Passwords **cannot be shared**.

# Exemplos em Contextos Profissionais

## Changelog

- Three bugs were fixed.
- A new dark mode feature was added.
- The loading speed was improved.

## Bug Report

- This error was reported by five users.
- The issue has been identified.
- A fix will be deployed tomorrow.

## Code Review

- This function should be renamed.
- The duplicated code must be removed.
- New tests were added to this branch.

# ATIVIDADE 1 - Active → Passive (20 min)

Reescreva o changelog abaixo da voz ativa para a voz passiva:

```
We fixed three bugs.
We added a new feature.
We removed the old menu.
We improved the loading speed.
We updated the security system.
We tested the new version.
```

Gabarito esperado (exemplo da primeira frase):
> Three bugs were fixed.

---

# ATIVIDADE 2 (JOGO) - Changelog Race (30 min)

**Como jogar:**

1. Dividir a turma em equipes de 3-4 alunos.
2. Cada equipe recebe um envelope/lista com 8 frases ativas sobre um app fictício (changelog completo).
3. Em 10 minutos, a equipe deve transformar o máximo de frases possível para a voz passiva, escrevendo no caderno ou no quadro.
4. Pontuação: +2 pontos por frase correta, +1 ponto bônus para a primeira equipe que terminar todas corretamente.
5. No final, cada equipe lê seu changelog em voz alta para a turma, como se fosse uma apresentação real de release notes.

Sugestão de frases para o envelope (o professor pode embaralhar/adaptar):

```
The team launched a new version.
Developers fixed the login error.
The designer improved the interface.
The QA team tested all features.
The company removed the old database.
Engineers deployed the update on Friday.
The support team solved the ticket.
The manager approved the new design.
```

# Leitura: Release Notes (20 min)

Leia o texto e responda às perguntas.

"Version 3.0 of the app was released last week. Several bugs were fixed, including the login error reported by many users. The dashboard was redesigned to be more intuitive, and a new dark mode feature was added. All user data is now encrypted before being stored. The update will be installed automatically on all devices."

**Perguntas de interpretação:**

1. O que foi corrigido nesta versão?
2. O que foi adicionado?
3. Como os dados dos usuários estão protegidos agora?
4. A atualização precisa ser instalada manualmente?

---

# ATIVIDADE 3 - Speaking em Pares (20 min)

Em pares, um aluno lê um "bug report" em voz alta (texto abaixo) e o outro identifica todas as frases na voz passiva, dizendo em voz alta qual é o sujeito da frase.

> "A critical error was found in the payment system. The bug was reported by three customers. The issue has been escalated to the development team. A temporary fix was applied this morning, and a permanent solution will be deployed by Friday."

---

# ATIVIDADE FINAL (JOGO) - Mystery Feature (25 min)

**Como jogar:**

1. Em grupos, cada equipe inventa uma funcionalidade nova para um app (sem dizer qual é!).
2. A equipe escreve 4-5 frases em **passive voice**, no estilo de release notes, descrevendo o que mudou — mas sem nomear a feature diretamente.

Exemplo:
> "Photos can now be shared with friends. New filters were added. The image quality was improved. Comments can be added below each post."
> (resposta: Instagram-like feature)

3. As outras equipes ouvem e tentam adivinhar qual feature foi descrita.
4. Ponto para quem adivinha certo, ponto bônus para a equipe que usou passive voice corretamente em todas as frases.
