# Inglês para Desenvolvedores — DO / DOES / DID

## 1. Introdução

Em inglês, **do**, **does** e **did** são **verbos auxiliares**.

Eles são usados para:

* Fazer **perguntas**
* Fazer **frases negativas**
* Dar **ênfase**
* Substituir o verbo principal em **respostas curtas**

Exemplos:

* Do you use Git?
  (Você usa Git?)

* Does the API return JSON?
  (A API retorna JSON?)

* Did the deployment fail?
  (O deploy falhou?)

---

# 2. Presente — DO / DOES

## Estrutura

| Sujeito | Auxiliar |
| ------- | -------- |
| I       | do       |
| You     | do       |
| We      | do       |
| They    | do       |
| He      | does     |
| She     | does     |
| It      | does     |

Regra simples:

**Do → I, You, We, They**
**Does → He, She, It**

Exemplos:

* I **do** write code every day.
  Eu realmente escrevo código todos os dias.

* Developers **do** test their applications.
  Desenvolvedores realmente testam suas aplicações.

* She **does** maintain the database.
  Ela mantém o banco de dados.

* The system **does** process requests.
  O sistema processa requisições.

---

# 3. Perguntas com DO / DOES

Estrutura:

**Do / Does + sujeito + verbo**

Exemplos na área de desenvolvimento:

* Do you write JavaScript?
  Você escreve JavaScript?

* Do developers use Git?
  Desenvolvedores usam Git?

* Do they deploy the application?
  Eles fazem deploy da aplicação?

* Does the server respond quickly?
  O servidor responde rapidamente?

* Does the API return JSON?
  A API retorna JSON?

* Does the system store passwords?
  O sistema armazena senhas?

Exemplo de diálogo:

Developer A:

> Do you use Docker?

Developer B:

> Yes, I do.

---

# 4. Frases negativas

Estrutura:

**Sujeito + do/does + not + verbo**

Contrações:

| Forma completa | Contração |
| -------------- | --------- |
| do not         | don't     |
| does not       | doesn't   |

Exemplos:

* I **don't use** PHP.
  Eu não uso PHP.

* We **don't deploy** on Friday.
  Nós não fazemos deploy na sexta.

* They **don't test** the code.
  Eles não testam o código.

* He **doesn't debug** the application.
  Ele não depura a aplicação.

* She **doesn't write** backend code.
  Ela não escreve código de backend.

* The server **doesn't respond**.
  O servidor não responde.

---

# 5. Respostas curtas (Short Answers)

Em inglês, é muito comum responder apenas com o **auxiliar**.

Exemplo:

Do you write Python?

Yes, I do.
No, I don't.

Outros exemplos:

Does the API work?

Yes, it does.
No, it doesn't.

Did the server crash?

Yes, it did.
No, it didn't.

---

# 6. Passado — DID

**Did** é o **passado de do e does**.

Ele é usado para:

* perguntas no passado
* negativas no passado
* dar ênfase no passado

Exemplos:

* Did you test the code?
  Você testou o código?

* Did the system fail?
  O sistema falhou?

* Did they deploy yesterday?
  Eles fizeram deploy ontem?

---

# 7. Perguntas com DID

Estrutura:

**Did + sujeito + verbo na forma base**

Exemplos:

* Did you push the code?
  Você fez push do código?

* Did the server restart?
  O servidor reiniciou?

* Did the deployment fail?
  O deploy falhou?

* Did the API return an error?
  A API retornou erro?

* Did the application crash?
  A aplicação travou?

⚠️ Importante:

Depois de **did**, o verbo volta para a **forma base**.

Exemplo:

❌ Did the server restarted?
✅ Did the server restart?

---

# 8. Negativa com DID

Estrutura:

**Sujeito + did not + verbo**

Contração:

**did not → didn't**

Exemplos:

* I didn't push the commit.
  Eu não fiz push do commit.

* We didn't deploy the application.
  Nós não fizemos deploy da aplicação.

* They didn't fix the bug.
  Eles não corrigiram o bug.

* The server didn't restart.
  O servidor não reiniciou.

* The system didn't save the data.
  O sistema não salvou os dados.

---

# 9. Ênfase com DO / DOES / DID

Esses auxiliares também podem dar **ênfase** a uma frase.

Exemplos:

* I **do understand** the problem.
  Eu realmente entendo o problema.

* She **does know** the solution.
  Ela realmente sabe a solução.

* We **did test** the system.
  Nós realmente testamos o sistema.

Isso é comum quando alguém está **corrigindo outra pessoa**.

Exemplo:

Manager:

> You didn't test the code.

Developer:

> I **did test** the code.

# 10. Verbos comuns em desenvolvimento

| Verbo     | Significado     |
| --------- | --------------- |
| write     | escrever código |
| build     | construir       |
| deploy    | fazer deploy    |
| test      | testar          |
| debug     | depurar         |
| create    | criar           |
| configure | configurar      |
| install   | instalar        |
| run       | executar        |
| compile   | compilar        |

Exemplos:

* Developers build software.
  Desenvolvedores constroem software.

* Engineers test systems.
  Engenheiros testam sistemas.

* Programmers debug code.
  Programadores depuram código.

* Teams deploy applications.
  Times fazem deploy de aplicações.

# 11. Exercício de leitura

Texto:

Developers write code every day.
They test their systems and deploy applications.
Good developers do review their code.
Yesterday they did deploy a new version of the API.

Perguntas:

1. Do developers write code?
2. Do they test their systems?
3. Did they deploy a new version yesterday?

---

## Exercício 2 — Complete com **did**

1. ___ you deploy yesterday?
2. ___ the server crash?
3. ___ they fix the bug?
4. ___ the API return an error?
5. ___ the system fail?

---

## Exercício 3 — Transforme em negativa

Exemplo:

We deploy on Friday → We **don't deploy** on Friday

1. Developers use SVN.
2. The API returns XML.
3. She writes backend code.
4. The system stores passwords.
5. The server processes requests.

---

## Exercício 4 — Negativa no passado

Exemplo:

They deployed → They **didn't deploy**

1. We tested the system.
2. The server restarted.
3. She fixed the bug.
4. They pushed the code.
5. The API responded.

---

## Exercício 5 — Transforme em pergunta

Exemplo:

You use Git → **Do you use Git?**

1. Developers use Docker.
2. The server runs Linux.
3. She writes documentation.
4. They deploy applications.
5. The system stores data.
